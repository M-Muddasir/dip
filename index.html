Chapter 1: Introduction

1.1 What is Digital Image Processing?

Digital Image: A two-dimensional function f(x, y) where x and y are spatial coordinates, and the amplitude of f at any point (x, y) is called the intensity or gray level. When x, y, and intensity values are all finite, discrete quantities → digital image.
Digital Image Processing: Processing digital images by means of a digital computer.

📌 Key Terms

Elements of a digital image are called: picture elements, image elements, pels, or pixels

Weber's Law

ΔI_c / I ≈ 0.02 (at moderate intensities)

Where: ΔI_c = just noticeable intensity difference, I = background intensity

Meaning: Eye can detect ~2% change in intensity

Important Visual Phenomena

Mach Bands

Definition: Apparent bright/dark bands at intensity transitions that don't physically exist

Cause: Lateral inhibition in retina

Significance: Perception ≠ Physical reality

Simultaneous Contrast

Definition: Same gray appears different against different backgrounds

Cause: Relative intensity processing

Significance: Absolute judgments unreliable

📌 RGB Key Points

  • Additive model: Adding colors creates lighter colors
  • Used in: Monitors, TVs, cameras, scanners
  • 8-bit per channel: 256 levels each → 256³ = 16.7 million colors
  • 24-bit color: 8 bits × 3 channels = 24 bits per pixel

HSI Color Model

Hue (H): The color attribute (0-360°) - red, green, blue, etc.
Saturation (S): Purity of color (0-1) - how vivid vs gray
Intensity (I): Brightness (0-1) - light vs dark

RGB to HSI Conversion

I = (R + G + B) / 3

S = 1 - [3 × min(R,G,B)] / (R+G+B)

H = cos⁻¹{[(R-G)+(R-B)] / [2√((R-G)²+(R-B)(G-B))]}

Color Image Storage

24-bit Color Image Storage

Storage = M × N × 24 bits = M × N × 3 bytes

Example: 1920×1080 color image = 1920 × 1080 × 3 = 6.2 MB

Rotation Transformation

Rotation Matrix (by angle θ counterclockwise)

┌─────────────────────┐
│ cos(θ)  -sin(θ)  0  │
│ sin(θ)   cos(θ)  0  │
│   0        0     1  │
└─────────────────────┘

Example for 30°: cos(30°) = 0.866, sin(30°) = 0.5

ROTATION EXAMPLE (30° counterclockwise): Original: Rotated: ┌─────────┐ ╱╲ │ │ ╱ ╲ │ IMAGE │ ───► ╱ IMAGE ╲ │ │ ╲ ╱ └─────────┘ ╲ ╱ ╲╱ Matrix: [0.866, -0.5, 0] [0.5, 0.866, 0] [0, 0, 1]

Projective (Perspective) Transformation

Projective Transformation Matrix

┌───────────────────────┐
│  a₁₁   a₁₂   a₁₃     │      [x']   [a₁₁x + a₁₂y + a₁₃]
│  a₂₁   a₂₂   a₂₃     │  →   [y'] = [a₂₁x + a₂₂y + a₂₃] / w
│  a₃₁   a₃₂    1      │      [w ]   [a₃₁x + a₃₂y + 1  ]
└───────────────────────┘

Cartesian coordinates: x' = x'/w,  y' = y'/w

⚠️ Key Difference: Affine vs Projective

  • Affine: Last row is [0, 0, 1] → w is always 1
  • Projective: Last row has non-zero a₃₁, a₃₂ → w varies, need to divide by w
  • Projective preserves: Only straight lines (not parallelism)

Forward vs Inverse Mapping

Method Direction Problem Solution
Forward Mapping Source → Destination Holes in output (unmapped pixels) Fill holes with interpolation
Inverse Mapping Destination → Source Non-integer source coordinates Interpolation (preferred method)

Composite Transformations

Order of Operations (Right to Left)

Tcomposite = Tn × Tn-1 × ... × T2 × T1

Important: Matrix multiplication is NOT commutative!
Scale then Rotate ≠ Rotate then Scale

Example: Rotate around a point (not origin)

To rotate an image around point (cx, cy) instead of origin:
Solution:
1. Translate point to origin: T(-cx, -cy)
2. Rotate by angle θ: R(θ)
3. Translate back: T(cx, cy)

Composite: T(cx,cy) × R(θ) × T(-cx,-cy)

# sm is now rotation arrayrix and homogeneous

tp = sm @ array([[rows],[cols],[1]]) # point is homogeneous

📖 EXPLAINED VERSION (With Line-by-Line Comments)

# =====================================================
# t2-rotation.py - IMAGE ROTATION (30 degrees)
# =====================================================

from PIL import Image                    # PIL: Python Imaging Library
from numpy import array, zeros           # NumPy for matrix operations
from matplotlib.pyplot import imshow, show, subplot  # For display

# STEP 1: Read image and convert to grayscale
img1 = Image.open('parinnda300.jpg').convert('L')  # 'L' = Luminance (grayscale)
im1 = array(img1)                          # Convert PIL image to numpy array

rows = im1.shape[0]                        # Get number of rows (height)
cols = im1.shape[1]                        # Get number of columns (width)

# STEP 2: Create rotation matrix for 30 degrees
# Formula: | cos(θ)  -sin(θ)  0 |
#          | sin(θ)   cos(θ)  0 |
#          |   0        0     1 |
# For 30°: cos(30°) = 0.866, sin(30°) = 0.5
sm = [[.866, -.5, 0],
      [.5, .866, 0],
      [0, 0, 1]]                           # Homogeneous rotation matrix

# STEP 3: Calculate output image size
# Transform corner point to find new dimensions
tp = sm @ array([[rows], [cols], [1]])     # @ = matrix multiplication
rowsd = int(tp[0,0]) + 200                # Add 200 for extra space
colsd = int(tp[1,0]) + 200                # (rotated image may be larger)

# STEP 4: Create empty output image (all zeros = black)
im2 = array(zeros([rowsd, colsd]))

# STEP 5: Forward mapping - transform each source pixel
r = 0                                     # Row counter
while r < rows:                           # Loop through all rows
    c = 0                                 # Column counter
    while c < cols:                       # Loop through all columns
        # Transform point [r, c, 1] using rotation matrix
        tp = sm @ array([[r], [c], [1]])  # Homogeneous point

        rd = int(tp[0,0])                # Destination row
        cd = int(tp[1,0])                # Destination column
        td = int(tp[2,0])                # td is ALWAYS 1 for affine!

        # Copy pixel if destination is valid (non-negative)
        if rd >= 0 and cd >= 0:
            im2[rd, cd] = im1[r, c]       # Copy pixel value

        c = c + 1                         # Next column
    r = r + 1                             # Next row

# STEP 6: Display original and rotated images
subplot(1, 2, 1)                          # 1 row, 2 cols, position 1
imshow(im1, cmap='gray')                  # Show original
subplot(1, 2, 2)                          # Position 2
imshow(im2, cmap='gray')                  # Show rotated
show()                                    # Display window

🔑 Key Concepts in Rotation Code

  • Homogeneous coordinates: [x, y, 1]ᵀ allows matrix multiplication for all transforms
  • Rotation matrix: [[cos θ, -sin θ, 0], [sin θ, cos θ, 0], [0, 0, 1]]
  • 30° rotation: cos(30°) = 0.866, sin(30°) = 0.5
  • Forward mapping: Source pixel → Destination pixel
  • td is always 1: For affine transforms, the w component doesn't change
  • While loops: Original uses while, can also use for loops

im1 = array(img1)

tp = sm @ array([[rows],[cols],[1]]) # point is homogeneous
rowsd = int(tp[0,0]/tp[2,0])+150 # added 150 to make more space
colsd = int(tp[1,0]/tp[2,0])+150

r = 0
while r < rows:
    c = 0
    while c < cols:
        tp = sm @ array([[r],[c],[1]]) # point is homogeneous
        rd = tp[0,0]
        cd = tp[1,0]
        td = tp[2,0]  # td is not 1 for projective

        rd = int(rd/td) # from homogeneous to cartesian
        cd = int(cd/td) # from homogeneous to cartesian
        td = int(td/td) # from homogeneous to cartesian

        if rd >=0 and cd >= 0:
            im2[rd, cd] = im1[r,c]

        c = c + 1
    r = r + 1

# =====================================================
# t3-projection.py - PROJECTIVE (PERSPECTIVE) TRANSFORM
# =====================================================

from PIL import Image
from numpy import array, zeros
from matplotlib.pyplot import imshow, show, subplot

# STEP 1: Read image
img1 = Image.open('parinnda300.jpg').convert('L')
im1 = array(img1)

rows = im1.shape[0]
cols = im1.shape[1]

# STEP 2: Create PROJECTIVE transformation matrix
# KEY DIFFERENCE: Last row is NOT [0, 0, 1]!
# Last row [0.0028, 0.0036, 1] makes this PROJECTIVE
sm = array([[.8, .3, 10],           # Row 1: affects x'
            [-.7, .9, 30],          # Row 2: affects y'
            [0.0028, 0.0036, 1]])  # Row 3: affects w (NOT [0,0,1]!)

# STEP 3: Calculate output size
# MUST divide by w component (tp[2,0]) for projective!
tp = sm @ array([[rows], [cols], [1]])
rowsd = int(tp[0,0] / tp[2,0]) + 150  # Divide by w!
colsd = int(tp[1,0] / tp[2,0]) + 150  # Divide by w!

# STEP 4: Create empty output image
im2 = array(zeros([rowsd, colsd]))

# STEP 5: Forward mapping with projective transform
r = 0
while r < rows:
    c = 0
    while c < cols:
        # Transform point
        tp = sm @ array([[r], [c], [1]])

        # Get homogeneous coordinates
        rd = tp[0,0]  # x' in homogeneous
        cd = tp[1,0]  # y' in homogeneous
        td = tp[2,0]  # w - THIS IS NOT 1 for projective!

        # CRITICAL: Convert homogeneous → Cartesian by dividing by w
        rd = int(rd / td)  # x_cartesian = x_homogeneous / w
        cd = int(cd / td)  # y_cartesian = y_homogeneous / w
        td = int(td / td)  # w becomes 1 after division

        # Copy pixel if valid
        if rd >= 0 and cd >= 0:
            im2[rd, cd] = im1[r, c]

        c = c + 1
    r = r + 1

# STEP 6: Display
subplot(1, 2, 1)
imshow(im1, cmap='gray')
subplot(1, 2, 2)
imshow(im2, cmap='gray')
show()

# STEP 7: Save result to file
img2 = Image.fromarray(im2.astype('uint8'), 'L')  # Convert array to image
img2.convert('RGB').save('my.jpg', 'jpeg')         # Save as JPEG

🔑 Key Concepts in Projective Code

  • Projective matrix: Last row has NON-ZERO values [0.0028, 0.0036, 1]
  • w varies: Unlike affine where w=1 always, projective w changes!
  • Homogeneous → Cartesian: MUST divide by w: x' = x'/w, y' = y'/w
  • Effect: Creates perspective distortion (like viewing at an angle)
  • Preserves: Only straight lines (parallelism is NOT preserved)

⚡ Key Difference: Rotation vs Projection

Aspect Rotation (Affine) Projection (Projective)
Matrix last row [0, 0, 1] [0.0028, 0.0036, 1]
w (td) value Always = 1 Varies (must divide!)
Need to divide by w? No (optional) YES (required!)
Preserves Parallelism, ratios Only straight lines
DOF 6 8

Comparison: Affine vs Projective

Aspect Affine Projective
Last row [0, 0, 1] [a₃₁, a₃₂, 1] (non-zero)
w value Always 1 Varies (must divide)

📝 Practice Problems

Problem 1: Storage Calculation

Calculate the storage required for a 1024×1024 grayscale image with 256 levels.
Solution:
k = log₂(256) = 8 bits per pixel
Storage = M × N × k
        = 1024 × 1024 × 8 bits
        = 8,388,608 bits
        = 1,048,576 bytes
        = 1 MB

Problem 2: Distance Calculation

Calculate De, D₄, and D₈ between p=(2,5) and q=(7,2).
Solution:
D_e = √[(7-2)² + (2-5)²] = √[25 + 9] = √34 ≈ 5.83

D₄ = |7-2| + |2-5| = 5 + 3 = 8

D₈ = max(|7-2|, |2-5|) = max(5, 3) = 5

Problem 3: Noise Reduction

An image has noise with σ = 30. How many images must be averaged to reduce noise to σ ≤ 6?
Solution:
σ_avg = σ / √K
6 = 30 / √K
√K = 30/6 = 5
K = 25 images

Problem 4: Wavelength Calculation

Calculate the wavelength of AC power at 60 Hz.
Solution:
λ = c / ν
λ = (3 × 10⁸ m/s) / (60 Hz)
λ = 5 × 10⁶ m
λ = 5000 km

Problem 5: Transformation

Apply rotation by 90° counterclockwise to point P=(4, 3).
Solution:
Rotation 90°: cos(90°) = 0, sin(90°) = 1

[x']   [0  -1  0] [4]   [0×4 + (-1)×3 + 0]   [-3]
[y'] = [1   0  0] [3] = [1×4 + 0×3 + 0   ] = [4 ]
[1 ]   [0   0  1] [1]   [0 + 0 + 1       ]   [1 ]

Result: P' = (-3, 4)
